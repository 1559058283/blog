# aitechflow.cn 个人博客 - 设计文档

## 1. 项目概述

- **项目名称**: aitechflow.cn 个人博客
- **类型**: 静态博客网站 + 评论系统
- **核心功能**: 技术/生活博客，支持评论和行内批注
- **目标用户**: 读者、技术爱好者

## 2. 视觉设计

### 配色方案
- **背景色**: 纯黑 `#000000`
- **文字色**: 纯白 `#ffffff`
- **次级文字**: 灰色 `#888888`
- **分割线**: 深灰 `#333333`
- **悬浮效果**: 白色 `#ffffff` (20% 透明度)

### 字体
- **主字体**: JetBrains Mono, SF Mono, Consolas, monospace
- **中文字体**: Noto Sans SC, system-ui

### 风格
- 极简黑白
- 无多余装饰
- 像高端技术文档
- 安静、专注的阅读体验

## 4. 功能需求

### 4.1 评论系统
- **技术方案**: Giscus (基于GitHub Discussions)
- **功能**:
  - 底部评论区
  - 文中批注/脚注
- **数据存储**: GitHub Discussions
- **无需数据库**

### 4.2 浏览量统计
- **技术方案**: 不使用后端数据库
- **方案A**: Firebase (免费额度足够)
- **方案B**: 不蒜子 (国内CDN)
- **备选**: 跳过此功能，首版先不上

### 4.3 时间显示
- 日期格式: YYYY-MM-DD
- 显示在文章标题下方

### 4.4 部署
- **源码存储**: GitHub 仓库
- **静态文件**: 部署到用户云服务器
- **部署方式**: 手动上传 或 CI/CD

## 5. 技术栈

- **生成器**: Hexo (静态博客框架)
- **主题**: 自定义 (基于 frontend-design 风格)
- **评论**: Giscus
- **代码托管**: GitHub
- **服务器**: 用户云服务器 (aitechflow.cn)

## 6. 数据备份

- 文章源码 (.md 文件): GitHub 仓库
- 评论数据: GitHub Discussions
- 部署配置: GitHub 仓库
- **结论**: 即使服务器故障，数据在 GitHub 完全备份
